Arquitetura de segurança
de nível institucional.

Projetado para operar sob os padrões regulatórios do sistema financeiro brasileiro. Dados do banco são o ativo mais sensível — a arquitetura reflete isso.

Dados nunca saem do Safra
Vector DB (Qdrant) e embedding model rodam on-premise no servidor do Safra. O LLM externo (Anthropic) recebe apenas o trecho relevante da pergunta — nunca o documento completo.

DPA com Anthropic: dados não são usados para treino, retenção zero após processar.
PII Blocking — Três Camadas
1
Na ingestão: documentos com CPF/CNPJ vão para collections restritas automaticamente
2
Na query: trechos sensíveis redactados antes de compor o prompt
3
Na resposta: middleware intercepta qualquer vazamento antes de publicar no grupo
Audit Trail Imutável
SHA-256 hash-chain, append-only. Impossível alterar retroativamente.

Retenção 5 anos (Res. BCB 4.658/2018). Registro completo: quem perguntou, quais documentos foram consultados, hash da resposta gerada.
Conformidade Regulatória
LGPD implementado (export + forget por titular).
SOC 2 Type I em preparação (Q3 2026).
Compatível com Resolução CMN 4.893 (Bacen — Segurança Cibernética).

Humano é sempre o checkpoint final — Joseph prepara, o gerente decide.
